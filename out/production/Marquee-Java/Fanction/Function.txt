context of the fun... (gloabl scope,local,...this...)
Local Scope → Variable inside function/block, usable only there.
Global Scope → Variable inside class, accessible by all methods.
Parameter Scope → Function input variables, usable only inside function.
this keyword → Refers to current object.
Block Scope → Variable inside { }, usable only in that block.

//call stack or heap memory(lvt,how obj store,stack frame,which is fast,consistence)
Stack Memory → Stores local variables & function calls, very fast, auto deleted.
Heap Memory → Stores objects & arrays, slower, managed by Garbage Collector.
LVT → Local Variable Table inside stack frame.
Stack Frame → Created for every function call, stores method data.
Object Storage → Reference in stack, actual object in heap.
Faster → Stack is faster than heap.
Consistency → Stack is fixed/organized, heap is dynamic/flexible.


//pass by value or pass by reference...
Pass by Value → Copy of variable is passed. Original value does not change.
Pass by Reference → Address/reference is passed. Original data can change.
